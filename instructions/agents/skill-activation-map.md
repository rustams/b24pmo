# Skill Activation Map

## Product/Stack Skills
- `navigate-b24-project`: use for locating files and entrypoints.
- `manage-b24-environment`: use for docker/make/env operations.
- `develop-b24-frontend`: use for Nuxt/B24UI implementation.
- `develop-b24-python`: use for Django/API integration.
- `implement-b24-features`: use for placement, events, robots.
- `bitrix24-static-local-app`: use for static local app packaging.
- `bitrix24-project-ops`: use for Bitrix24 task operations (status + kanban sync), roadmap metadata sync, and post-push VPS deploy verification.

## Context-Engineering Skills
- `context-fundamentals`: when designing context boundaries.
- `context-degradation`: when failures appear in long sessions.
- `context-compression`: when history grows and needs compaction.
- `context-optimization`: when reducing token cost / context noise.
- `filesystem-context`: when offloading state/outputs to files.
- `multi-agent-patterns`: when splitting tasks across agents.
- `memory-systems`: when designing persistent memory/state.
- `tool-design`: when defining tools or API contracts for agents.
- `evaluation`: for baseline quality evaluation.
- `advanced-evaluation`: for LLM-as-judge and bias mitigation.
- `project-development`: for phased implementation plans.
- `hosted-agents`: for remote/background agent execution patterns.
- `bdi-mental-states`: for cognitive-state modeling (optional, R&D).

## Activation Policy
1. Always activate the smallest set of skills that can solve the task.
2. Prefer stack/product skills first.
3. Add context-engineering skills only when task complexity requires them.
4. Persist decisions and artifacts after each significant step.
