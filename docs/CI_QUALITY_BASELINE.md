# RD-002: Базовый контур качества CI

## Цель
Зафиксировать минимально необходимый набор автоматических проверок, которые запускаются одинаково локально и в CI.

## Что входит в baseline
- Проверка синтаксиса Python-кода backend (`compileall`).
- Проверка валидности ключевых JSON-файлов roadmap.
- Линт frontend через ESLint.

## Точки входа
- Локально: `make quality-check`
- Скрипт: `scripts/quality-check.sh`
- CI: `.github/workflows/quality-baseline.yml`

## Критерии готовности RD-002
- В репозитории есть единый quality-скрипт.
- В `makefile` добавлена команда `quality-check`.
- В GitHub Actions добавлен workflow, который запускает baseline на `push` и `pull_request` в `master`.
- Результаты проверок воспроизводимы локально.

## Ограничения и оговорки
- Полноценные unit/integration тесты будут добавляться по мере реализации доменов roadmap.
- Текущий baseline является минимальным и ориентирован на раннее выявление регрессий в структуре и синтаксисе.
