# Смарт-процесс «Цели» — структура полей

Документ описывает набор полей для смарт-процесса **«Цели»**, предназначенного для стратегического планирования в рамках проектного офиса (PMO Hub). Поля сгруппированы по функциональным блокам для удобства навигации.

## 1. Основная информация

| Название поля | Код поля | Тип | Сортировка | Обязательное | Множественное | Описание |
|---------------|----------|-----|------------|---------------|----------------|----------|
| **Название цели** | `UF_CRM_XXX_GOAL_TITLE` | Строка | 10 | Да | Нет | Краткое наименование цели |
| **Описание цели** | `UF_CRM_XXX_GOAL_DESCRIPTION` | Текст | 20 | Нет | Нет | Развёрнутое описание цели |
| **Тип цели** | `UF_CRM_XXX_GOAL_TYPE` | Привязка к элементам инф. блоков | 30 | Да | Нет | Категория цели (стратегическая, операционная, финансовая и т.п.) |
| **Приоритет** | `UF_CRM_XXX_GOAL_PRIORITY` | Привязка к элементам инф. блоков | 40 | Нет | Нет | Важность цели (низкий, средний, высокий, критический) |
| **Статус цели** | `UF_CRM_XXX_GOAL_STATUS` | Привязка к элементам инф. блоков | 50 | Да | Нет | Текущее состояние (в работе, достигнута, не достигнута, отменена) |

## Примечания

- **Код поля** (например, `UF_CRM_XXX_...`) является шаблонным. При создании смарт-процесса в Битрикс24 будет сгенерирован уникальный префикс (`UF_CRM_174...`), который заменяет `XXX`.
- **Сортировка** задаёт порядок полей в карточке; значения можно корректировать под конкретные нужды.
- Для справочников (типы целей, приоритеты, статусы, единицы измерения) необходимо создать соответствующие **инфоблоки** или использовать списки Битрикс24.
- Поле **«Ключевой показатель (KPI)»** сделано множественным, чтобы можно было указать несколько показателей для одной цели (например, несколько метрик).
- Поля **«Ключевые риски»** и **«Меры минимизации»** выделены как обязательные, чтобы стимулировать проработку рисков на этапе постановки цели.
